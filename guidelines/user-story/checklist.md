# User Story Validation Checklist

## INVEST

| Criterion | Check |
|-----------|-------|
| **I**ndependent | Can it be delivered standalone? No hard external blockers? |
| **N**egotiable | Is there room for refinement and discussion? |
| **V**aluable | Does it deliver clear end-user or business value? |
| **E**stimable | Is it defined well enough to estimate effort? |
| **S**mall | Can it fit within a single sprint? |
| **T**estable | Are acceptance criteria explicit and verifiable? |

## 3Cs

| Element | Check |
|---------|-------|
| **C**ard | Is the narrative written clearly? (As a… I want… so that…) |
| **C**onversation | Are collaborators, context, and open questions documented? |
| **C**onfirmation | Are the acceptance tests / conditions of satisfaction defined? |

## Consistency Checks

- [ ] The "who" (role) is a real user of the system
- [ ] The "what" (action) is a single, focused capability
- [ ] The "why" (benefit) justifies the effort
- [ ] All acceptance criteria are objectively pass/fail
- [ ] No hidden assumptions or implicit requirements
- [ ] Dependencies are identified and externalized
